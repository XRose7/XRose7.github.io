LeetCode 1171 Remove Zero Sum Consecutive Nodes from Linked List

Posted on 2024-03-12 In LeetCode

Remove Zero Sum Consecutive Nodes from Linked List - LeetCode

Notes

一题一解：前缀和 + 哈希表 - ylb

若链表节点的两个前缀和相等，说明两个前缀和之间的连续节点序列的和为 0 ，那么可以消去这部分连续节点.

从链表中删去总和值为零的连续节点 - 题解评论 - Edward Elric

但删除也带来了一定问题，那就是删除掉的一系列节点的区间和需要“作废”，否则后续你再次尝试查找区间和时，可能会查到已经被删除的节点对应的区间和，产生“幽灵”一般的效果。所以最初遇见这题，想要做单次遍历，想到的很自然是在需要删除时，顺便删除掉所有对应的哈希表项（间接导致我需要记录下每个节点对应的前缀和）

这样就不会有“幽灵区间”的问题了，但是代码看起来不大好看了，复杂度方面，由于删除的节点最多就n个，所以时间复杂度仍然是O(n)。而题解则是用了一种更加巧妙的做法：

  • 第一遍遍历，记录前缀和和节点，如果有多个相同的前缀和，则保留最后一个（这个很重要）
  • 第二遍遍历，第一次遇到相同前缀和，则直接进行删除（删掉第一次和最后一次相同前缀和的整个序列）
  • 第一遍遍历保留最后一个，确保了第二遍总是会向后查找，而非向前查找，也就不需要去动态维护 map 里的元素了

Codes

Time: O(n)

Space: O(n)

LeetCode 791 Custom Sort String

Posted on 2024-03-11 In LeetCode

Custom Sort String - LeetCode

Thinkings

自定义比较函数排序.

Codes

Time: O(mlogm + k)

Space: O(k)

m is the length of s

k is a constant value: 26

sort() 's space complexity is O(mlogm)

Notes

自定义字符串排序 - 力扣官方题解

Better custom sort

Time: O(mlogm + k)

Space: O(k)

m is the length of s

k is a constant value: 26

sort() 's space complexity is O(mlogm)

Counting sort

由于字符集的大小为 26 ，我们也可以考虑使用计数排序代替普通的排序方法.

我们首先遍历字符串 s ，使用数组或哈希表统计每个字符出现的次数. 随后遍历字符串 order 中的每个字符 c ，如果其在 s 中出现了 k 次，就在答案的末尾添加 k 个 c ，并将数组或哈希表中对应的次数置为 0 . 最后我们遍历一次哈希表，对于所有次数 k 非 0 的键值对 (c, k) ，在答案的末尾添加 k 个 c 即可.

Time: O(n + k)

Space: O(k)

m is the length of s

k is a constant value: 26

LeetCode 2540 Minimum Common Value

Minimum Common Value - LeetCode

Thinkings

类似于归并排序的双指针遍历比较. 当然也可以直接用 hash-table 遍历两数组，最后得出结果.

Codes

Time: O(n + m)

Space: O(1)

Notes

Binary search approach

由于两个数组都按升序排序，因此对于数组 \(\textit{nums}_1\)中的每个元素 \(\textit{num}\) ，可以使用二分查找判断 \(\textit{num}\) 是否在数组 \(\textit{nums}_2\) 中出现，在两个数组中都出现的最小元素即为答案。如果数组 \(\textit{nums}_1\) 中的所有元素都不在数组 \(\textit{nums}_2\) 中出现，则答案是 −1 .

由于两个数组都按升序排序，因此可以从左向右遍历数组 \(\textit{nums}_1\) 中的每个元素，找到的第一个在两个数组中都出现的元素即为答案.
